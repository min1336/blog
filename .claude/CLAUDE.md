# 프로젝트 규칙

## 코딩 규칙
- 한국어 주석 사용
- API 키, 비밀값 하드코딩 절대 금지
- debug/print 로그 커밋 금지
- 불필요한 변경 금지 — 요청된 것만 구현
- 기존 코드의 스타일, 패턴, 컨벤션을 분석하고 그대로 따를 것

## 구현 후 필수 검증
코드베이스를 분석하여 해당 프로젝트의 도구로 검증하세요:
1. 테스트 실행 — 프로젝트의 테스트 프레임워크 사용 (package.json, Makefile, pyproject.toml 등에서 확인)
2. 린트/포맷 — 프로젝트에 설정된 린터/포매터 실행 (있는 경우에만)
3. 타입 체크 — 프로젝트에 타입 시스템이 있으면 실행 (있는 경우에만)
4. 빌드 — 빌드 스크립트가 있으면 빌드 성공 확인

## 커밋 컨벤션
- feat: 새 기능
- fix: 버그 수정
- refactor: 리팩토링
- test: 테스트 추가/수정
- docs: 문서
- style: 포맷팅 (코드 변경 없음)
- chore: 빌드/설정

## PR 규칙
- 제목: 한국어, conventional commits, 50자 이내
- 본문: 변경 요약 + Closes #이슈번호 + 테스트 방법
